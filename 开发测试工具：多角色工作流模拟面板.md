 2. dev-test.js 路由

 GET /api/dev/test-accounts（无需认证）
 - 返回所有测试账号信息：name、phone、roles、password（明文，仅测试环境）

 POST /api/dev/quick-login（无需认证）
 - body: { role: 'DESIGN' }
 - 找到对应测试账号，生成 JWT，返回 token + userInfo
 - 用于面板一键切换角色，不走密码验证

 POST /api/dev/run-scenario（需 ADMIN token）
 - body: { scenario: 'full' | 'cross_area' | 'reject_resubmit' | 'timeout_remind' }
 - 服务端用各测试角色的身份依次调用业务 API，记录每步结果
 - 返回：
 {
   "scenario": "full",
   "steps": [
     { "step": "创建工单", "status": "pass", "data": { "demandNo": "DM-20260406-001" } },
     { "step": "设计提交", "status": "pass", "data": {} },
     ...
   ],
   "passed": 5,
   "failed": 0
 }
 - 测试完成后自动清理本次创建的测试工单（不清理测试用户）

 3. TestPanelView.vue

 布局分三区：

 区域 A：测试账号
 - 表格展示所有测试账号（角色、姓名、手机、密码）
 - 每行有「复制凭据」按钮（复制为 phone: xxx / password: xxx 格式）
 - 「一键获取 Token」按钮：调用 /api/dev/quick-login，将 token+userInfo 存入当前 admin-web 的 localStorage（仅 ADMIN
 角色可用），对其他角色弹窗展示 token 便于在 miniprogram 或 Postman 中使用

 区域 B：场景测试
 - 四个场景卡片，各有「运行」按钮（调用 /api/dev/run-scenario）
 - 运行中显示 loading，完成后展示步骤列表（✅ / ❌ + 耗时）

 区域 C：快速说明
 - 如何在小程序中使用测试账号（手机号+密码登录）
 - 如何用 Postman 调试（复制 token 的方式）
