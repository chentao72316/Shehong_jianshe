# 飞书端配置步骤

项目：通信运营商一线支撑小程序
文档版本：V1.0
日期：2026-03-26

---

## 项目现状说明

### 已实现
- MongoDB 作为主数据库（非飞书多维表格）
- `feishu.js` 实现了 Webhook 群消息发送
- 超时检查服务调用飞书提醒
- User 模型有 `feishuId` 字段（用于 @mention）

### 尚未实现的飞书集成
- 9 种消息场景只实现了超时相关 2 种
- 消息卡片无法 @人（Webhook 机器人无法获取用户 open_id）
- 新需求/驳回/设计完成/施工完成的飞书通知未触发
- 人员配置表的飞书同步未实现

## 第一步：创建飞书群

1. 飞书 → 新建群聊
2. 群名称：**一线建设支撑群**
3. 将以下人员加入群：所有网格经理、网络支撑经理、设计/施工/监理单位负责人

---

## 第二步：创建自定义机器人（Webhook）

> 用于快速实现基础消息推送，但**无法 @指定用户**

1. 进入群聊 → 右上角 `···` → **群设置** → **群机器人** → **添加机器人**
2. 选择**自定义机器人**
3. 机器人名称：`建设需求助手`，上传图标
4. 安全设置选择**签名校验**（推荐）或 IP 白名单
5. 复制生成的 **Webhook URL**

将 URL 填入服务端环境变量：

```
FEISHU_WEBHOOK_URL=https://open.feishu.cn/open-apis/bot/v2/hook/xxxxxxxx
FEISHU_WEBHOOK_SECRET=xxxxxxxx   # 如果开启签名
```

---

## 第三步：创建飞书开放平台应用（必须，才能实现 @人）

> 自定义 Webhook 机器人无法 @指定用户，必须通过**开放平台应用机器人**实现

1. 进入飞书开放平台（https://open.feishu.cn/app）→ **创建企业自建应用**
2. 应用名称：`建设需求助手`，描述填写用途
3. 进入应用 → **添加应用能力** → **机器人**
4. 进入**权限管理**，开通以下权限：

| 权限 | 用途 |
|------|------|
| `im:message:send_as_bot` | 机器人发送消息 |
| `im:chat` | 读取群信息 |
| `contact:user.phone:readonly` | 查询用户手机号（用于认证匹配） |
| `contact:user.id:readonly` | 获取用户 open_id（用于 @mention） |

5. 发布应用版本，等待管理员审核通过
6. 记录 **App ID** 和 **App Secret**

```
FEISHU_APP_ID=cli_xxxxxxxxxxxxxxxx
FEISHU_APP_SECRET=xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

7. 将该应用机器人加入**一线建设支撑群**：群设置 → 机器人 → 添加机器人 → 搜索应用名称

---

## 第四步：收集所有人员的飞书 ID

> 用于填入人员配置表的 `feishuId` 字段，实现 @mention

### 手动获取方式（管理员逐个操作）

1. 飞书 → 管理后台（https://admin.feishu.cn）
2. 通讯录 → 搜索每位相关人员 → 点击人员详情 → 复制 **User ID**（格式：`ou_xxxxxxxx`）

### 批量获取方式（推荐，通过 API）

```bash
# 用手机号批量查询 feishuId
POST https://open.feishu.cn/open-apis/contact/v3/users/batch_get_id
Authorization: Bearer {tenant_access_token}
Content-Type: application/json

{
  "phones": ["13800138001", "13800138002"],
  "user_id_type": "open_id"
}
```

将获取到的 `open_id` 填入管理后台（调用 `/api/admin/staff/save` 接口的 `feishuId` 字段）。

---

## 第五步：获取群的 Chat ID

> 发送消息到群需要群的 `chat_id`

机器人加入群后，调用接口获取群列表：

```bash
GET https://open.feishu.cn/open-apis/im/v1/chats
Authorization: Bearer {tenant_access_token}
```

记录群的 `chat_id`，填入环境变量：

```
FEISHU_CHAT_ID=oc_xxxxxxxxxxxxxxxx
```

---

## 第六步：验证 Webhook 是否正常

临时测试（直接 curl）：

```bash
curl -X POST "你的WEBHOOK_URL" \
  -H "Content-Type: application/json" \
  -d '{
    "msg_type": "interactive",
    "card": {
      "header": {
        "title": {"tag": "plain_text", "content": "测试消息"},
        "template": "blue"
      },
      "elements": [{
        "tag": "div",
        "text": {"tag": "lark_md", "content": "飞书集成配置验证"}
      }]
    }
  }'
```

群内收到消息则配置成功。

## 环境变量完整模板

在 `server/` 目录下创建 `.env` 文件：

```env
# 服务
PORT=3000
JWT_SECRET=your_secret_here
JWT_EXPIRES_IN=7d

# 数据库
MONGODB_URI=mongodb://localhost:27017/shehong_jianshe

# 微信小程序
WX_APPID=wx_your_appid
WX_SECRET=your_wx_secret

# 飞书（Webhook 机器人）
FEISHU_WEBHOOK_URL=https://open.feishu.cn/open-apis/bot/v2/hook/xxx
FEISHU_WEBHOOK_SECRET=

# 飞书（开放平台应用）
FEISHU_APP_ID=cli_xxx
FEISHU_APP_SECRET=xxx
FEISHU_CHAT_ID=oc_xxx
```

## 配置完成后需补充开发的代码

配置好飞书后，以下代码缺口需要同步开发：

### 1. `server/src/utils/feishu.js`
增加通过开放 API 发送带 @mention 的消息（当前只有 Webhook）。

### 2. 业务路由中补充飞书通知触发

| 文件 | 位置 | 触发场景 |
|------|------|----------|
| `routes/demand.js` | `POST /demand/create` | 场景1：新需求录入 |
| `routes/design.js` | 驳回操作 | 场景2：需求驳回 |
| `routes/design.js` | 设计提交 | 场景3：设计完成 |
| `routes/construction.js` | 施工提交 | 场景4：施工完成 |

### 3. `services/timeout-checker.service.js`
补充以下预警场景（当前只实现了超时后状态变更）：

- 设计 > 1.5 天 → 场景5：设计即将超时预警
- 施工 > 4 天 → 场景7：施工紧急督办
